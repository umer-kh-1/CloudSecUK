Invoke-WebRequest

Invoke-WebRequest -Uri https://www.youtube.com
$Response = Invoke-WebRequest -Uri https://www.youtube.com

$Response | Get-Member  ## to view all available properties

$Response.StatusCode  ## to view the StatusCode property

Here’s a detailed explanation of its components:

1. Status Information

  • StatusCode: 200This means the server successfully processed the request and returned content.
  • StatusDescription: OKA textual description of the status code, confirming the success.

2. Content

  • HTML Structure:

    The response contains a basic HTML document starting with <!DOCTYPE html>. It includes:

    • Language and Direction: <html lang="en-GB" dir="ltr"> specifies English (British) as the language and left-to-right text direction.
    • CSS Styling: Includes inline styles (e.g., link colors, font family) with a security nonce to prevent unauthorized script injection.
  • Key Visual Elements:
    • Image:An <IMG> tag references YouTube’s logo with multiple resolutions for different displays (e.g., standard and high-resolution srcset images).
    • Sign-In Button:A clickable link (<A> tag) labeled Sign in points to a Google login page for user authentication.

3. Metadata and Security Headers

  • Headers:Various HTTP headers enforce security and resource policies:
    • Pragma: no-cacheInstructs browsers not to cache the response.
    • Content-Security-Policy: Specifies allowed sources for scripts, objects, and other content. Includes:
      • script-src: Limits script execution to specific sources with nonce.
      • object-src: Prevents the use of plugins or embedded objects.
      • report-uri: Specifies a reporting endpoint for violations.
      • worker-src: Restricts worker scripts to the same origin.
    • Cross-Origin-*: Policies governing resource sharing and browser interactions between origins.

4. Input Fields

Hidden input fields within the HTML form carry metadata:

  • Example:
    • <INPUT type="hidden" name="bl" value="boq_identityfrontenduiserver_20241210.00_p0">This suggests backend metadata for maintaining session or form state.
    • Fields like gl=GB (geolocation) and x=8 (possibly an internal identifier) are also present.

5. Links

  • Anchor Tag (<A>):A key link redirects users to Google’s authentication service with parameters like:
    • hl=en-GB (language).
    • continue=https://www.youtube.com/shorts/... (redirect after login).
  • SVG Icon:The Sign in link includes an inline SVG image for a user-friendly icon.

6. Raw HTTP Details

  • RawContent: Shows the HTTP version (HTTP/1.1), headers, and raw response data.
  • Forms: Structured data representing any forms on the page.
